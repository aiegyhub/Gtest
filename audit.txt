### **File Audit: `public/robots.txt`**

The `robots.txt` file instructs web crawlers which pages or files they can or cannot request from a site. A well-configured `robots.txt` is essential for crawl efficiency and preventing the indexing of sensitive or duplicate content.

#### **1. Technical SEO Audit**

*   **Sitemap:** `Sitemap: https://saudiservices.sa/sitemap.xml`
    *   **Critical Issue:** The domain `saudiservices.sa` is incorrect. Based on our previous analysis, the brand is "توب كلينرز" (Top Cleaners) and the domain should be `https://top-cleaners.net`. This must be corrected.
*   **Disallowed Directories:**
    *   `Disallow: /admin` and `/admin/*`: **Excellent.** This correctly blocks crawlers from accessing the admin panel.
    *   `Disallow: /api/*`: **Good practice.** This prevents crawlers from making API calls.
    *   `Disallow: /*.json$`: This is a good general rule to prevent indexing of JSON files, though it might be redundant with `/api/*` depending on the setup. It doesn't cause harm.
*   **Allowed Directories:**
    *   `Allow: /`, `Allow: /sa`, `Allow: /sa/*`, `Allow: /services/*`, etc.: These `Allow` rules are mostly redundant because the default behavior for `User-agent: *` is to allow everything unless explicitly disallowed. While they don't cause harm, they add unnecessary clutter. The only time `Allow` is truly needed is to make an exception within a disallowed directory (e.g., `Allow: /admin/login` inside a `Disallow: /admin/`).
*   **Asset Rules:** `Allow: /assets/*`, `Allow: /*.css`, `Allow: /*.js`, etc.: **This is a critical best practice.** Allowing crawlers to access CSS and JS files is essential for Google to render pages correctly and understand the user experience. This is well-implemented.
*   **Bot-Specific Rules:**
